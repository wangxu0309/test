分布式版本控制系统：服务器保存文件的所有更新版本，客户端是服务器的完整备份，并不是保留文件的最新版本；优点：联网运行，支持多人开发，断网后支持离线本地提交版本更新，服务器损坏后可使用任何一个客户端的备份进行恢复
使用git管理项目，拥有三个区域，工作区、暂存区、Git仓库区
Git中的三种状态，已修改(modified)、已暂存(staged)、已提交(committed)
配置用户信息git config --global user.name/user.email,运行一次永久生效，会被写入c/用户/王旭/.gitconfig用记事本打开就可以查到全局配置信息，运行终端命令来检查全局配置信息 git config --list --global查看指定的全局配置项 git config user.name
安装并配置GIT获取帮助信息：git help 如git help config直接在浏览器中打开，终端打印 git config -h
获取Git仓库的两种方式
  • 将尚未进行版本控制的本地目录转换为Git仓库
  • 从其他服务器克隆一个已经存在的Git仓库
git.init命令会创建一个名为.git的隐藏目录，这个.git隐藏目录就是当前的git仓库，里面包含了初始化的必要文件，这些文件是Git仓库的必要组成部分
工作区中文件的四种状态：未被管理、已修改、未修改、已暂存用git status（简单的git status -s或git status -short）查看文件处于什么状态，未被跟踪的文件Git不会自动纳入跟踪范围,跟踪新文件git add +文件名，添加多个文件git add . ;git commit -m "提交消息" 提交到git仓库中
modified已跟踪的文件内容发生了变化还未被放到暂存区git add：开始跟踪新文件；把已跟踪已修改的文件放到暂存区；把有冲突的文件标记为已解决状态
撤销对文件的修改：把工作区中对应文件的修改，还原成git仓库中保存的版本，操作结果：所有修改会丢失，危险性比较高，git checkout --git.html
跳过使用暂存区域 git commit -a -m "描述消息"
取消暂存的文件 git reset HEAD 要移除的文件名
从git仓库中移除文件有两种方式：从git仓库和工作区同时移除对应文件；从git仓库中移除指定的文件git rm -f 文件名称，但是在工作区保留相应的文件git rm --cached 文件名
